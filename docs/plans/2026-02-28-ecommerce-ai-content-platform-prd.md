# 电商 AI 内容生成与数据分析平台 — 产品需求文档（PRD）

> 版本：v1.0
> 日期：2026-02-28
> 状态：已确认

---

## 一、项目背景

本系统是一个多租户 B2B SaaS 电商智能客服平台，已具备完整的对话管理、知识库 RAG 检索、模型配置、拼多多平台对接等能力。

为进一步提升平台价值，计划新增 **AI 内容生成、电商平台深度集成、智能数据分析** 三大方向共 6 个功能模块，帮助商家实现从「商品管理 → 内容创作 → 智能运营 → 数据洞察」的全链路赋能。

---

## 二、目标用户

- **电商商家（租户）**：在拼多多、淘宝、京东、抖音等平台经营店铺的中小商家
- **运营人员**：负责商品上架、内容制作、数据分析的店铺运营

---

## 三、功能概览

| 编号 | 功能模块 | 核心能力 | 阶段 |
|------|---------|---------|------|
| F1 | 商品海报生成 | 文生图 / 图生图 | 阶段 2 |
| F2 | 商品宣传视频生成 | 文生视频 / 图生视频 | 阶段 2 |
| F3 | 一键上传到平台 | 素材推送到电商平台 / 关联商品 | 阶段 2 |
| F4 | 商品信息自动同步 | 平台商品同步到系统 + 写入知识库 | 阶段 1 |
| F5 | 订单分析报告 | 统计图表 + AI 解读 + 下载 | 阶段 4 |
| F6 | 智能标题/描述/定价 | AI 生成文案 + 竞品定价分析 | 阶段 3 |

### 交付优先级

```
阶段 1 → 阶段 2 → 阶段 3 → 阶段 4
```

阶段 1 是所有功能的数据基础，必须最先交付。

## 四、功能详细需求

### F4 — 商品信息自动同步（阶段 1）

#### 4.1 功能描述

用户对接电商平台后，系统自动拉取其店铺的所有商品数据，存储到本地，并将商品信息格式化写入知识库，使客服 AI 能够自动检索商品信息回答买家咨询。

#### 4.2 用户故事

- 作为商家，我希望对接平台后系统**自动获取我的全部商品**，无需手动录入
- 作为商家，我希望商品信息**自动同步到知识库**，这样客服 AI 就能回答买家关于商品的问题
- 作为商家，我希望系统能**定时同步商品变更**（价格调整、上下架等），保持数据最新
- 作为商家，我希望能**查看同步状态和历史**，了解同步是否成功

#### 4.3 功能要求

**4.3.1 商品同步**

| 需求项 | 描述 |
|--------|------|
| 全量同步 | 用户对接平台后，一次性拉取所有在售商品 |
| 增量同步 | 定时拉取最近变更的商品（新增/修改/下架） |
| 同步间隔 | 用户可配置，默认 60 分钟，最小 10 分钟 |
| 手动触发 | 用户可随时手动触发全量或增量同步 |
| 冲突处理 | 同一时间只允许一个同步任务运行 |
| 同步记录 | 记录每次同步的状态、数量、耗时、失败原因 |

**4.3.2 商品数据字段**

| 字段 | 说明 | 必需 |
|------|------|------|
| 商品标题 | 平台侧商品名称 | 是 |
| 商品描述 | 详情描述 | 否 |
| 当前售价 | 实际销售价格 | 是 |
| 原价 | 划线价 | 否 |
| 商品分类 | 所属类目 | 否 |
| 商品图片 | 主图 + 轮播图 URL 列表 | 否 |
| 商品视频 | 视频 URL 列表 | 否 |
| SKU 属性 | 颜色、尺码等规格 | 否 |
| 销量 | 累计销量 | 否 |
| 库存 | 当前库存 | 否 |
| 商品状态 | 在售 / 下架 / 已删除 | 是 |

**4.3.3 知识库集成**

- 每个商品同步后自动生成一条知识库条目（类型：PRODUCT）
- 知识库内容包含：商品名称、价格、分类、描述、规格、库存、销量
- 商品更新时自动更新对应知识库条目
- 知识库条目自动触发向量化，支持 RAG 检索

**4.3.4 多平台支持**

- 设计为多平台通用接口，通过适配器模式扩展
- 首期实现拼多多平台商品同步
- 后续可扩展淘宝、京东、抖音等平台

#### 4.4 页面设计

**商品管理页 `/products`**

- **统计概览**：商品总数、在售商品数、最近同步时间
- **操作区**：全量同步按钮、增量同步按钮、同步调度配置入口
- **同步进度**：正在同步时展示进度条（已同步/总数）
- **商品列表**：表格展示，支持搜索（标题）、筛选（状态/分类）、分页
- **商品详情弹窗**：展示完整商品信息 + 知识库关联状态
- **同步调度配置弹窗**：开关 + 间隔时间设置

### F1 — 商品海报生成（阶段 2）

#### 1.1 功能描述

用户可以通过选择商品或手动输入信息，结合提示词模板，调用多模态大模型生成商品宣传海报。支持文生图和图生图两种模式。

#### 1.2 用户故事

- 作为商家，我希望能**一键生成商品海报**，不需要 PS 技能
- 作为商家，我希望能**选择不同风格的模板**（简约、促销、节日等）
- 作为商家，我希望能**上传参考图进行二次创作**（图生图）
- 作为商家，我希望能**批量生成多张候选海报**，从中挑选最满意的
- 作为商家，我希望生成的海报能**直接下载或上传到电商平台**

#### 1.3 功能要求

**1.3.1 模型配置**

- 在现有模型配置系统中新增「图像生成」模型类型
- 用户在设置中配置图像生成模型的 provider、API Key、模型名称
- 支持的 provider 包括但不限于：通义万相（qwen）、智谱 CogView（zhipuai）等

**1.3.2 提示词模板系统**

| 需求项 | 描述 |
|--------|------|
| 系统内置模板 | 预置多种风格模板（简约、促销、节日、品类专用等） |
| 自定义模板 | 用户可创建自己的模板 |
| 变量占位符 | 模板支持 `{{商品名}}` `{{价格}}` `{{卖点}}` 等变量 |
| 模板分类 | 按品类（服装/数码/食品）和风格分类 |
| 效果预览 | 模板可附带预览图 |

**1.3.3 文生图**

- 用户选择商品（自动填充变量）或手动输入信息
- 选择提示词模板或自定义提示词
- 设置生成参数：尺寸（方形/横版/竖版）、生成数量（1~4张）
- 提交后异步生成，完成后通知用户
- 展示生成结果，支持下载、收藏、上传到平台

**1.3.4 图生图**

- 用户上传参考图片或选择商品已有图片
- 输入修改指令（如「换成红色喜庆背景」「添加促销文案」）
- 系统将参考图 + 提示词组合发送给模型
- 后续流程同文生图

#### 1.4 页面设计

**海报工作台 `/content/poster`**

- **左侧面板**：
  - 商品选择器（从已同步商品中选择，自动填充信息）
  - 模板选择器（网格展示，支持分类筛选）
  - 提示词编辑区（可编辑模板内容或自定义）
  - 参考图上传区（图生图模式）
  - 生成参数设置（尺寸、数量）
  - 「生成」按钮
- **右侧面板**：
  - 生成结果展示（图片网格）
  - 每张图片下方操作：下载 / 收藏 / 上传到平台
- **底部**：生成历史记录列表

### F2 — 商品宣传视频生成（阶段 2）

#### 2.1 功能描述

用户可以通过文字描述或上传图片，调用多模态大模型生成商品宣传短视频。

#### 2.2 用户故事

- 作为商家，我希望能**用文字描述生成产品展示视频**
- 作为商家，我希望能**用商品图片生成动态视频**（图生视频）
- 作为商家，我希望能**设置视频参数**（时长、分辨率、风格）
- 作为商家，我希望视频生成完成后能**收到通知**（因为生成较慢）

#### 2.3 功能要求

**2.3.1 模型配置**

- 在模型配置系统中新增「视频生成」模型类型
- 支持的 provider：智谱 CogVideoX、通义等

**2.3.2 文生视频**

- 用户输入视频描述或使用提示词模板
- 设置参数：时长（3s / 5s / 10s）、分辨率（720p / 1080p）、风格（写实/动画/产品展示）
- 异步生成，WebSocket 通知完成
- 支持预览播放、下载、上传到平台

**2.3.3 图生视频**

- 用户上传图片或选择商品图片作为起始帧
- 输入运动描述（如「相机环绕商品 360 度旋转」）
- 后续流程同文生视频

**2.3.4 特殊处理**

- 视频生成耗时较长（30 秒 ~ 数分钟），需要：
  - 异步任务队列处理
  - 实时进度通知
  - 对于异步 API 的轮询支持
- 生成的视频文件存储到对象存储

#### 2.4 页面设计

**视频工作台 `/content/video`**

- 布局同海报工作台，增加：
  - 视频参数配置面板（时长/分辨率/风格）
  - 视频预览播放器
  - 生成进度指示器（因耗时较长）

### F3 — 一键上传到平台 / 关联商品（阶段 2）

#### 3.1 功能描述

用户生成的海报和视频可以一键上传到对接的电商平台，并与指定商品关联。

#### 3.2 用户故事

- 作为商家，我希望生成的海报能**一键上传到我的电商平台店铺**
- 作为商家，我希望能**选择上传到哪个商品**下作为商品图/视频
- 作为商家，我希望能在**素材库中统一管理**所有生成的内容
- 作为商家，我希望能看到每个素材**是否已上传到平台**的状态

#### 3.3 功能要求

**3.3.1 上传到平台**

| 需求项 | 描述 |
|--------|------|
| 选择平台 | 用户选择目标电商平台（已对接的） |
| 关联商品 | 选择要关联的商品 |
| 上传图片 | 将生成的海报上传为商品主图或轮播图 |
| 上传视频 | 将生成的视频上传为商品视频 |
| 上传状态 | 展示上传结果（成功/失败/平台侧URL） |

**3.3.2 素材库**

| 需求项 | 描述 |
|--------|------|
| 统一管理 | 所有生成的图片和视频集中展示 |
| 筛选 | 按商品、类型（图片/视频）、时间、收藏状态筛选 |
| 批量操作 | 支持批量下载、批量删除 |
| 收藏 | 用户可收藏优质素材 |
| 平台状态 | 展示每个素材在各平台的上传状态 |

#### 3.4 页面设计

**素材库 `/content/assets`**

- **顶部**：筛选区（商品/类型/时间/收藏）
- **主体**：素材网格/列表视图切换
  - 每个素材卡片展示：缩略图、商品名、生成时间、平台上传状态
  - 操作：预览、下载、上传到平台、收藏、删除
- **批量操作栏**：多选后展示批量操作按钮

### F6 — 智能标题 / 描述生成 + 智能定价（阶段 3）

#### 6.1 功能描述

**文案生成**：用户选择商品的图片或视频，系统通过多模态大模型分析视觉内容，自动生成商品标题和描述文案。

**智能定价**：系统收集平台内同类商品数据（脱敏）和用户上传的竞品数据，结合 AI 分析给出定价建议。

#### 6.2 用户故事

**文案生成：**
- 作为商家，我希望能**根据商品图片自动生成标题**，不用费脑想文案
- 作为商家，我希望系统能**生成多个候选标题和描述**供我选择
- 作为商家，我希望能**选择文案风格**（简洁专业/促销吸睛/SEO 优化）
- 作为商家，我希望满意的文案能**一键应用到商品**

**智能定价：**
- 作为商家，我希望系统能**推荐合理的商品定价**
- 作为商家，我希望能看到**同类商品的价格分布**
- 作为商家，我希望能**上传竞品信息**来辅助定价分析
- 作为商家，我希望能看到 **AI 的定价理由和建议**

#### 6.3 功能要求

**6.3.1 文案生成**

| 需求项 | 描述 |
|--------|------|
| 多模态输入 | 支持图片和视频作为分析素材 |
| 标题生成 | 生成 3~5 个候选标题 |
| 描述生成 | 生成 2~3 个候选描述 |
| 风格选择 | 简洁专业 / 促销吸睛 / 卖点突出 / SEO 优化 |
| 平台适配 | 根据目标平台的标题长度限制自动调整 |
| 一键应用 | 选定文案后可直接更新商品标题/描述（通过平台 API） |

**6.3.2 智能定价**

| 需求项 | 描述 |
|--------|------|
| 竞品数据源 — 平台内 | 基于向量相似度检索本平台所有租户的同类商品（脱敏处理，仅暴露价格/分类/销量） |
| 竞品数据源 — 用户上传 | 支持手动录入或 CSV/Excel 批量导入竞品信息 |
| 统计分析 | 竞品均价、中位数、价格区间、价格分布 |
| AI 定价建议 | 推荐价格 + 价格区间 + 文字建议 |
| 可视化 | 价格分布直方图 / 散点图 |

**6.3.3 数据脱敏规则**

- 平台内跨租户查询时：仅暴露价格、分类、销量
- 不暴露：商品标题、店铺名称、租户信息
- 排除本租户自身的商品

#### 6.4 页面设计

在**商品详情页**中以 Tab 形式嵌入：

**文案生成 Tab**
- 商品图片/视频选择区
- 风格选择下拉框
- 「生成文案」按钮
- 候选标题列表（可一键复制/应用）
- 候选描述列表（可一键复制/应用）

**智能定价 Tab**
- 竞品数据概览表格
- 上传竞品入口（手动录入 + CSV 导入）
- 定价分析结果卡片：推荐价格 + 建议区间 + AI 分析文字
- 价格分布可视化图表

### F5 — 订单分析报告（阶段 4）

#### 5.1 功能描述

系统对接电商平台订单数据，结合商品信息进行销售分析，以统计图表 + AI 智能解读的形式生成分析报告，支持 PDF 和 Excel 下载。

#### 5.2 用户故事

- 作为商家，我希望系统能**自动同步我的订单数据**
- 作为商家，我希望能看到**直观的销售数据仪表盘**（图表）
- 作为商家，我希望能**生成一份完整的分析报告**，包含 AI 对数据的解读和经营建议
- 作为商家，我希望报告支持**下载为 PDF 或 Excel**，方便分享和存档
- 作为商家，我希望能按**时间范围**和**报告类型**来生成报告

#### 5.3 功能要求

**5.3.1 订单同步**

- 复用商品同步架构（全量 + 定时增量）
- 订单数据字段：订单号、商品、买家、数量、单价、总金额、订单状态、支付时间、发货时间、完成时间、退款金额

**5.3.2 分析报告类型**

| 类型 | 内容 |
|------|------|
| 销售报告 | 整体销售数据、营收趋势、客单价 |
| 商品报告 | 热销/滞销排行、品类分析 |
| 趋势报告 | 日/周/月销量趋势、同比环比 |
| 综合报告 | 以上全部 + AI 经营建议 |

**5.3.3 报告内容**

| 内容项 | 数据来源 |
|--------|---------|
| 关键指标 | 总订单数、总营收、客单价、退款率 |
| 销量趋势 | 每日销量折线图 + AI 解读 |
| 热销商品 | Top N 商品柱状图 + AI 分析原因 |
| 品类占比 | 饼图 + 占比表格 |
| 退款分析 | 退款率趋势 + 高退款商品预警 |
| AI 经营建议 | 基于全部数据的 3~5 条可执行建议 |

**5.3.4 报告导出**

| 格式 | 内容 |
|------|------|
| PDF | 封面 + 图表 + AI 分析 + 数据附录 |
| Excel | 订单明细 + 各维度统计表 |

**5.3.5 报告生成流程**

1. 用户选择时间范围 + 报告类型
2. 系统异步执行：SQL 统计 → 图表生成 → AI 分析 → 文件导出
3. 完成后通知用户，可在线预览或下载

#### 5.4 页面设计

**订单概览 `/analytics/orders`**
- 关键指标卡片（总订单/营收/客单价/退款率）
- 时间筛选器
- 订单列表（搜索/筛选/分页）

**分析报告 `/analytics/reports`**
- 生成报告入口：选择时间范围 + 报告类型 + 「生成」按钮
- 历史报告列表（状态/生成时间/下载）
- 报告在线预览页（图表 + AI 分析文字 + 下载按钮）

**销售看板 `/analytics/dashboard`**
- 实时图表仪表盘
- 热销商品排行
- 品类销售占比饼图
- 销量趋势折线图

## 五、非功能需求

### 5.1 性能

| 指标 | 要求 |
|------|------|
| 商品同步 | 全量同步 1000 件商品 < 5 分钟 |
| 图片生成 | 单张图片 < 30 秒 |
| 视频生成 | 5 秒短视频 < 3 分钟 |
| 报告生成 | 含 AI 分析的综合报告 < 2 分钟 |
| 页面加载 | 列表页首屏 < 2 秒 |

### 5.2 多平台扩展性

- 所有平台相关功能通过适配器模式实现
- 新增平台只需实现适配器接口，无需修改业务逻辑
- 首期支持拼多多，架构预留淘宝、京东、抖音扩展

### 5.3 多租户数据隔离

- 每个租户只能访问自己的商品、订单、素材、报告数据
- 智能定价中跨租户查询需严格脱敏
- 租户间数据通过 tenant_id 逻辑隔离

### 5.4 异步任务

- 所有耗时操作（同步、生成、报告）均异步执行
- 通过 WebSocket 或轮询通知用户任务完成
- 任务支持查看状态、失败重试

## 九、关键技术决策

| 决策项 | 选择 | 原因 |
|--------|------|------|
| AI 生成方式 | 调用多模态大模型，通过现有模型配置系统管理 | 复用已有架构，用户可灵活选择 provider |
| 电商平台对接 | 适配器模式，统一抽象接口 | 支持多平台扩展，低耦合 |
| 商品同步策略 | 全量 + 定时增量 | 数据完整性 + 实时性平衡 |
| 报告生成 | 统计图表 + AI 解读 | 数据可信（图表）+ 洞察价值（AI） |
| 定价数据来源 | 平台内租户数据（脱敏）+ 用户上传竞品 | 数据丰富度 + 隐私合规 |
| 整体架构 | 功能模块独立扩展 | 与现有架构一致，无迁移成本 |
